# 학과 과제 압박 랭킹 프론트 구조 계획

## Summary
- 프론트는 `Next.js App Router` 기반 모바일 웹 MVP로 만든다. 기본 기준 화면은 390px 모바일이고, 데스크톱은 가운데 정렬된 모바일 캔버스로 처리한다.
- 최신 기준은 `학과 과제 압박 랭킹 서비스 MVP 계획서`와 `docs/week1` 문서다. 오래된 교수 중심 `PLAN.md` 내용은 프론트 구현 기준에서 제외한다.
- 공개 단위는 오직 `학교명 + 학과명`이다. 교수, 강의, 과목, 댓글, 리뷰, 즐겨찾기, 관리자 CMS, AI 마스코트 생성은 만들지 않는다.
- 핵심 행동은 `로그인 -> 학교 선택 -> 학과 검색 -> 상세 진입 -> 압박 올리기`이며, 로그인 이후 검색에서 클릭까지 3탭 이내로 유지한다.

## Functional Structure
- 인증/사용자 상태
  - 사용자 상태는 `GUEST`, `AUTH_NO_SCHOOL`, `ACTIVE_USER` 3개로 고정한다.
  - `/`는 공개 진입 화면이고, `Google로 시작하기` CTA만 제공한다.
  - `/home`, `/departments/*`, `/me/*`는 로그인 필요 화면이다.
  - 로그인했지만 학교를 선택하지 않은 사용자는 `/onboarding/school`로 보낸다.
  - 공유 링크 `/departments/:id?ref=share`는 비로그인 상태에서 학과 미리보기와 `로그인 후 계속` CTA를 보여주고, 로그인 후 원래 상세로 복귀한다.

- 학교 선택
  - `/onboarding/school`에서 학교 목록을 검색하고 하나를 선택한다.
  - 선택한 학교는 칩으로 표시하고 `학교 선택 완료` 버튼을 누르면 사용자 프로필의 `selected_school_id`를 저장한 뒤 `/home`으로 이동한다.
  - 학교 선택 이후 홈과 검색 화면의 기본 학교 필터는 사용자의 선택 학교다.

- 홈 랭킹
  - `/home`은 `전국 학과 압박 TOP 10`을 중심으로 구성한다.
  - 상단에는 학교 필터 드롭다운과 검색 아이콘을 둔다.
  - 랭킹 카드는 약 72px 높이로 `순위`, `학과명`, `학교명`, `totalClicks`, `stackCount`, `pressureLevel`을 보여준다.
  - `오늘 급상승`은 가로 캐러셀로 제공한다.
  - 보호 화면에는 하단 탭바 `홈`, `검색`, `내 활동`을 고정한다.

- 학과 검색/등록
  - `/departments/search`는 학교 필터 칩, 검색 입력, 결과 카드, 빈 상태를 가진다.
  - 검색 입력 placeholder는 `학과 검색`으로 고정한다.
  - 결과가 없거나 원하는 학과가 없을 때 `결과가 없나요?`와 `새 학과 등록` CTA를 보여준다.
  - `/departments/new`는 현재 선택 학교를 잠금 상태로 표시하고, 학과명 입력, 8개 카테고리 선택, 템플릿 미리보기, `등록하고 바로 시작` CTA를 제공한다.
  - 등록 시 프론트에서도 학과명 기본 정규화 규칙을 적용해 사용자에게 중복 가능성을 미리 보여주되, 최종 중복 판단은 API 응답을 따른다.
  - `DUPLICATE` 응답이면 신규 생성하지 않고 기존 학과 상세로 이동시키는 안내를 보여준다.

- 학과 상세/클릭
  - `/departments/:id`는 `OO대학교 컴퓨터공학과` 형식의 제목, 마스코트 카드, 압박 단계 배지, 총 클릭 수, 학생 스택 수, 오늘 클릭 수를 보여준다.
  - 마스코트 카드는 최소 220px 높이이며, 카테고리 템플릿과 `pressureLevel`에 따라 표정, 배경 강도, 스택 표현만 바꾼다.
  - 하단 thumb zone에 가장 강한 CTA인 `압박 올리기` 버튼을 fixed로 둔다.
  - 클릭 성공 시 서버 응답의 `newTotalClicks`, `stackCount`, `pressureLevel`로 화면 수치를 즉시 갱신하고 300ms 숫자 상승 피드백을 준다.
  - `accepted=false`는 클릭 이벤트는 접수됐지만 랭킹 합산 제외된 상태로, 버튼 실패처럼 보이지 않게 짧은 안내 토스트로 처리한다.
  - `공유하기`는 현재 학과의 `?ref=share` 링크를 복사하거나 Web Share API를 사용한다.

- 내 활동
  - `/me/activity`는 `내 최근 클릭`, 오늘 클릭 수, 최근 클릭 목록을 보여준다.
  - 각 행은 학교명, 학과명, 클릭 시간, accepted 여부를 표시한다.
  - `다시 누르기` CTA는 해당 학과 상세로 이동시키고, 상세 화면에서 클릭을 유도한다.

## Implementation Structure
- 라우트 구조
  - `app/page.tsx`: 랜딩
  - `app/onboarding/school/page.tsx`: 학교 선택
  - `app/home/page.tsx`: 홈
  - `app/departments/search/page.tsx`: 학과 검색
  - `app/departments/new/page.tsx`: 학과 등록
  - `app/departments/[id]/page.tsx`: 학과 상세 및 공유 진입
  - `app/me/activity/page.tsx`: 내 활동
  - `app/auth/callback/route.ts`: Google OAuth 콜백 처리

- 도메인 모듈
  - `features/auth`: Supabase 세션, 사용자 상태 판정, 로그인/로그아웃
  - `features/schools`: 학교 검색, 선택 저장, 학교 필터 UI
  - `features/rankings`: 랭킹 조회, 랭킹 카드, 급상승 카드
  - `features/departments`: 검색, 등록, 상세 조회, 학과 정규화, 카테고리 템플릿
  - `features/clicks`: 클릭 요청, optimistic feedback, rate-limit 안내
  - `features/activity`: 내 최근 클릭 조회
  - `features/share`: 공유 링크 생성, 공유 진입 상태 처리

- 공통 UI
  - `AppShell`: 모바일 캔버스, 안전 영역, 보호 화면 레이아웃
  - `TopAppBar`: 학교 필터, 검색 진입
  - `BottomTabBar`: `홈`, `검색`, `내 활동`
  - `RankingCard`, `DepartmentResultCard`, `MascotCard`, `PressureBadge`, `ScoreNumber`, `PrimaryButton`, `SecondaryButton`, `Toast`
  - 디자인 토큰은 문서의 색상, 폰트, 간격, radius를 그대로 CSS variables로 둔다.
  - 모든 visible copy는 한국어만 사용한다.

- API 연결
  - `GET /api/rankings?scope=national|school&schoolId=`
  - `GET /api/departments/search?q=&schoolId=&limit=20`
  - `POST /api/departments`
  - `GET /api/departments/:id`
  - `POST /api/departments/:id/click`
  - `GET /api/me/activity?limit=20`
  - API 에러는 `UNAUTHORIZED`, `SCHOOL_NOT_SELECTED`, `NOT_FOUND`, `DUPLICATE`, `RATE_LIMITED`, `VALIDATION_ERROR`, `INTERNAL_ERROR` 기준으로 화면별 토스트와 리다이렉트를 처리한다.

## UI Rules
- 스타일 콘셉트는 `campus arcade dashboard`로 고정한다.
- 주요 색상은 `--brand: #c45012`, 카드 배경 `#fffaf0`, 메인 배경 `#f7f4ed`를 사용한다.
- 압박 단계 색상은 `level0 #d8f3dc`, `level1 #ffe8a3`, `level2 #ffc078`, `level3 #ff922b`, `level4 #e03131`로 고정한다.
- 제목, 랭킹 숫자, 클릭 수는 `Space Grotesk`; 한국어 본문, 라벨, 버튼은 `Pretendard`를 사용한다.
- Primary button은 56px 높이, 배경 `#c45012`, 흰색 텍스트다.
- 카드 radius는 16px, 버튼 radius는 14px다.
- 마스코트는 실제 인물처럼 보이지 않는 계열별 템플릿 캐릭터만 사용한다.
- 마스코트 카테고리는 `공학`, `자연과학`, `인문`, `사회과학`, `경영/경제`, `예술/체육`, `교육`, `보건/의학` 8개다.

## Test Plan
- 비로그인 사용자는 `/`와 공유 미리보기만 접근 가능해야 한다.
- 로그인 후 학교 미선택 사용자는 보호 라우트 접근 시 `/onboarding/school`로 이동해야 한다.
- 학교 선택 완료 후 `/home`에서 전국 TOP 10과 오늘 급상승이 표시되어야 한다.
- 학과 검색은 `schoolId + q` 기준으로 결과를 보여주고, 결과 없음 상태에서 신규 등록 CTA가 보여야 한다.
- 신규 학과 등록 성공 시 즉시 해당 학과 상세로 이동해야 한다.
- 중복 학과 등록 응답을 받으면 기존 학과 상세로 안내해야 한다.
- 상세 화면에서 `압박 올리기` 클릭 후 총 클릭 수, 스택 수, 압박 단계가 API 응답 기준으로 갱신되어야 한다.
- 클릭 실패나 네트워크 실패 시 마지막 성공 수치를 유지하고 재시도 가능한 토스트를 보여야 한다.
- 공유 링크로 들어온 비로그인 사용자는 로그인 후 같은 학과 상세로 복귀해야 한다.
- 390px 모바일 기준으로 하단 고정 버튼과 탭바가 겹치지 않아야 한다.

## Assumptions
- 프론트 구현은 `Next.js App Router + Supabase Auth` 기준으로 진행한다.
- MVP에서는 별도 관리자 화면을 만들지 않고 Supabase 콘솔 운영을 전제로 한다.
- 오늘 급상승 API가 별도 확정되지 않았으므로 v1 프론트에서는 랭킹 API 또는 백엔드 제공 임시 집계 응답을 같은 카드 구조로 소비한다.
- 클릭 조작 판정은 프론트에서 하지 않고 서버 응답의 `accepted` 값을 그대로 반영한다.
- 학과명 정규화는 UX 보조용으로만 프론트에 두고, 최종 중복 방지는 서버와 DB 제약이 담당한다.
