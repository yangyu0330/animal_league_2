## 1. 주간 목표
- 팀장이 혼자 1주차를 진행해 MVP의 핵심 결정을 고정한다.
- 2주차부터 팀원 3명이 즉시 구현에 들어갈 수 있도록 설계 문서와 태스크를 완성한다.
- 1주차 종료 시점에 "미정 의사결정 0건"을 목표로 한다.

## 2. 이번 주 완료 조건 (Definition of Done)
- 기능 범위 확정 문서 1개 완료
- 화면 구조도 + 사용자 플로우 문서 1개 완료
- 모바일 와이어프레임 4개 화면 이상 완료
- DB 스키마 초안 1개 완료
- API 명세 초안 1개 완료
- 시드 데이터 설계 문서 1개 완료
- 2주차 개발 태스크 보드(프론트/백엔드/데이터) 작성 완료

## 3. 고정 의사결정 (변경 금지)
- 공개 단위: 학교명 + 학과명
- 랭킹 기준: 총 클릭 수
- 클릭 정책: 자유 클릭 허용, 오토클릭만 제외
- 인증: Google OAuth 1종
- 마스코트: 계열별 기본 템플릿
- 제외 기능: 교수/과목/댓글/게시판/즐겨찾기

## 6. 오토클릭 감지 MVP 기준
- 10초 내 동일 계정의 비정상 burst 클릭은 `accepted=false` 후보로 분류
- 계정 + 디바이스 해시 + IP 패턴이 동시에 과도할 때만 랭킹 제외
- 제외된 클릭도 `ClickEvent`에는 저장
- 감지 규칙 튜닝은 3주차 이후 로그 기반으로 조정

## 7. 리스크와 대응

- 대응: 정규화 사전 + 유사명 수동 병합 규칙 도입

- 리스크: 클릭 조작으로 랭킹 왜곡
- 대응: 오토클릭 제외 + 원본 로그 저장 + 주간 검토
