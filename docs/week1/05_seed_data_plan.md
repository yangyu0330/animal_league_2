# 시드 데이터 계획

## 목표
- 2주차에 바로 적재 가능한 학교/학과 시드 데이터를 정의한다.
- 초기 검색 체감이 나오도록 최소 데이터 볼륨을 확보한다.

## 입력 자료
- [01_scope.md](./01_scope.md)
- [05_department_normalization.md](./05_department_normalization.md)

## 시드 범위
- 대학: 20개
- 대학당 학과: 8~12개
- 예상 초기 학과 수: 200개 내외

## 대학 선정 기준
- 수도권/비수도권 균형
- 국립/사립 균형
- 검색 빈도가 높은 학교 우선

## CSV 스키마
```csv
school_name,region,is_seeded,department_name,department_normalized,category,template_id
```

## 작성 규칙
- `school_name`: 공식 학교명
- `region`: `수도권|충청|호남|영남|강원|제주`
- `department_name`: 사용자 노출명
- `department_normalized`: 정규화 키 (공백/특수문자 제거, 소문자)
- `category`: 8개 카테고리 중 1개
- `template_id`: 카테고리별 기본 템플릿 ID

## 데이터 수집/검수 흐름
1. 학교별 공개 학과 리스트 수집
2. 카테고리 매핑
3. 정규화 키 생성
4. 중복 검수
5. 샘플 20행 업로드 테스트
6. 전체 업로드
