# 와이어프레임

## 목표
- 모바일 우선 저해상도 설계로 2주차 UI 구현 착수 비용을 줄인다.
- 홈, 검색, 상세, 내 활동 4개 핵심 화면의 구조를 고정한다.

## 입력 자료
- [02_user_flow.md](./02_user_flow.md)
- [02_screen_map.md](./02_screen_map.md)

## 화면 1: 홈 (`/home`)
```text
┌──────────────────────────────┐
│ [학교 필터▼]   [검색 아이콘] │
│ 전국 학과 압박 TOP 10         │
│ 1. OO대 컴퓨터공학과 12,340   │
│ 2. XX대 경영학과     11,902   │
│ ...                           │
│ 오늘 급상승                   │
│ [카드][카드][카드]            │
│                              │
│ [홈] [검색] [내 활동]         │
└──────────────────────────────┘
```

## 화면 2: 학과 검색 (`/departments/search`)
```text
┌──────────────────────────────┐

│ [학과 검색 입력창           ] │
│ 결과 12건                    │
│ ┌카드: 컴퓨터공학과      >┐  │
│ ├카드: 전자공학과        >┤  │

│ [새 학과 등록]                │
│                              │
│ [홈] [검색] [내 활동]         │
└──────────────────────────────┘
```

## 공통 인터랙션
- 메인 클릭 버튼은 하단 고정형 대형 버튼(`thumb zone` 우선)
- 클릭 성공 시 숫자 상승 애니메이션 300ms
- 압박 단계 변경 시 마스코트 카드 색상과 표정 동시 변경
- 네트워크 실패 시 버튼 비활성 + 재시도 토스트
