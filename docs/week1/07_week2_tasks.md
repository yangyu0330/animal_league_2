# 2주차 태스크 분배

## 목표
- 2주차에 3명이 병렬로 작업할 수 있게 태스크를 분해한다.
- 각 태스크의 완료 조건을 명확히 해 리뷰 비용을 줄인다.

## 입력 자료
- [02_screen_map.md](./02_screen_map.md)
- [04_schema.md](./04_schema.md)
- [04_api_spec.md](./04_api_spec.md)

## 공통 원칙
- 우선순위는 `P0 > P1 > P2`
- P0가 끝나기 전 P2 착수 금지
- 모든 태스크는 PR 단위로 분리

## 팀원 A (프론트엔드)
- P0-1: App Router 기본 구조와 라우트 가드
  - 완료 기준: `/`, `/onboarding/school`, `/home`, `/departments/:id` 진입 가능
- P0-2: 홈 화면 구현
  - 완료 기준: 랭킹 리스트/학교 필터/급상승 카드 렌더링
- P1-1: 검색/등록 화면 구현
  - 완료 기준: 검색 결과, 등록 CTA, 빈 결과 상태 처리
- P1-2: 상세 화면 구현
  - 완료 기준: 클릭 버튼, 수치 카드, 공유 버튼 연결
- P2-1: 내 활동 화면 구현
  - 완료 기준: 최근 클릭 목록 20건 표시

## 팀원 B (백엔드)
- P0-1: Supabase 스키마 반영
  - 완료 기준: 5개 테이블 생성 + 인덱스 적용
- P0-2: 인증/프로필 연동
  - 완료 기준: 로그인 후 `app_user` 생성 및 학교 선택 저장
- P1-1: 랭킹/검색/상세 API 구현
  - 완료 기준: `GET /rankings`, `GET /departments/search`, `GET /departments/:id` 동작
- P1-2: 클릭 API + 오토클릭 판정 구현
  - 완료 기준: `POST /departments/:id/click`와 accepted 판정 동작
- P2-1: 내 활동 API 구현
  - 완료 기준: `GET /me/activity` 동작

## 팀원 C (풀스택/데이터)
- P0-1: 시드 CSV 초안 200행 작성
  - 완료 기준: 컬럼 규칙 준수, 중복 0건
- P0-2: 학과명 정규화 사전 적용 스크립트 작성
  - 완료 기준: 샘플 50건에서 정규화 결과 검수 통과
- P1-1: 마스코트 템플릿 자산 정리
  - 완료 기준: 카테고리 8종 기본 자산 준비
- P1-2: 이벤트 분석(PostHog) 기본 이벤트 정의
  - 완료 기준: `login`, `search`, `click`, `share` 이벤트 명세 완료
- P2-1: QA 시나리오 문서화
  - 완료 기준: 핵심 시나리오 12개 작성

## 통합 일정 (2주차)
- 월: P0 착수
- 화: P0 마감
- 수: P1 착수
- 목: P1 마감
- 금: 통합/버그수정/P2 일부

## 의존성
- 프론트 상세 화면은 백엔드 `GET /departments/:id`가 먼저 필요
- 클릭 UX 검증은 `POST /departments/:id/click` 완료 후 진행
- 시드 데이터 업로드는 스키마 반영 완료 후 진행
