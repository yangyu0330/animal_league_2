# 사용자 흐름

## 목표
- 로그인 이후 핵심 행동인 `학과 검색 -> 상세 진입 -> 클릭`을 3탭 이내로 고정한다.
- 신규 학과 등록과 공유 유입 흐름을 분리해 예외 경로를 명확히 한다.

## 사용자 상태
- `GUEST`: 로그인 전
- `AUTH_NO_SCHOOL`: 로그인 완료, 학교 미선택
- `ACTIVE_USER`: 로그인 완료, 학교 선택 완료

## 플로우 A: 첫 방문 사용자
1. `GUEST`가 `/` 진입
2. 서비스 설명과 샘플 랭킹 확인
3. `Google 로그인` 클릭
4. 로그인 성공 시 `/onboarding/school` 이동
5. 학교 선택 후 `/home` 이동
6. 홈에서 랭킹 카드 또는 검색 진입

## 플로우 B: 기존 사용자 클릭
1. `/home` 진입
2. 검색창에서 학과 검색
3. 결과 리스트에서 학과 선택
4. `/departments/:id` 이동
5. `압박 올리기` 버튼 클릭
6. 클릭 결과 토스트 및 수치 갱신 확인

## 플로우 C: 학과 신규 등록
1. 검색 결과 없음 확인
2. `새 학과 등록` 클릭
3. 학교 자동 채움 + 학과명 입력 + 계열 선택
4. 중복 검사 통과 시 생성 API 호출
5. 생성 직후 상세 페이지로 리다이렉트
6. 최초 클릭 유도 CTA 노출

## 플로우 D: 공유 링크 유입
1. 외부 공유 링크 `/departments/:id?ref=share` 진입
2. 로그인 상태면 바로 상세 노출
3. 비로그인 상태면 로그인 후 원래 상세로 복귀
4. 클릭 완료 시 `공유 유입`으로 이벤트 기록

## 예외 플로우
- 학교 미선택 상태에서 상세 직접 진입 시 학교 선택 화면으로 이동
- 등록 중 중복 감지 시 기존 학과 상세 링크 제시
- 클릭 API 실패 시 재시도 버튼과 마지막 성공 수치 유지
- 삭제되거나 차단된 학과 ID 진입 시 `/home`으로 폴백
