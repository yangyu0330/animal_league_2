# 화면 구조도

## 목표
- MVP에 필요한 화면만 남기고 각 화면의 책임을 1개씩 명확히 고정한다.
- 화면별 라우트, 핵심 CTA, 필요한 API를 매핑해 2주차 구현 착수 문서로 사용한다.

## 화면 목록
| 화면 | 라우트 | 진입 조건 | 핵심 CTA | 필요 데이터 |
|---|---|---|---|---|
| 랜딩 | `/` | 누구나 | `Google로 시작하기` | 샘플 랭킹 5개 |
| 학교 선택 | `/onboarding/school` | 로그인 완료 + 학교 미선택 | `학교 선택 완료` | 학교 목록 |
| 홈 | `/home` | 로그인 + 학교 선택 완료 | `학과 검색`, `랭킹 카드 탭` | 전국 TOP10, 오늘 급상승, 학교별 TOP5 |

| 학과 등록 | `/departments/new` | 검색 결과 없음 또는 직접 진입 | `등록하고 바로 시작` | 학교 기본값, 계열 목록 |
| 학과 상세 | `/departments/:id` | 로그인 + 유효한 학과 ID | `압박 올리기`, `공유` | 학과 기본정보, 클릭 수, 압박 단계 |
| 내 활동 | `/me/activity` | 로그인 | `다시 누르기` | 내 최근 클릭 20개 |
| 공유 랜딩 | `/departments/:id?ref=share` | 외부 유입 | `로그인 후 계속` | 학과 기본정보 |

## 라우팅 가드 규칙
- 비로그인 사용자가 `/home` 이하 진입 시 `/`로 리다이렉트
- 학교 미선택 사용자가 `/departments/*`, `/me/*` 진입 시 `/onboarding/school`로 리다이렉트
- 존재하지 않는 학과 ID는 `/home`으로 리다이렉트하고 `학과를 찾을 수 없음` 토스트 노출

## 화면별 고정 컴포넌트
- 전역: 상단 앱바, 하단 고정 탭바(`홈`, `검색`, `내 활동`)
- 홈: 랭킹 카드 리스트, 학교 필터 드롭다운, 급상승 캐러셀
- 검색: 학교 필터 칩, 검색 입력창, 결과 카드, 등록 유도 패널
- 상세: 마스코트 카드, 수치 패널, 메인 클릭 버튼, 공유 버튼
