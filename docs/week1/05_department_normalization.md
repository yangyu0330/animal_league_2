# 학과명 정규화

## 목표
- 사용자 등록 학과와 시드 학과의 중복을 최소화한다.
- `school_id + normalized_name` 기준 유니크 정책이 실제 동작하도록 규칙을 고정한다.

## 입력 자료
- [05_seed_data_plan.md](./05_seed_data_plan.md)
- [04_schema.md](./04_schema.md)

## 정규화 규칙 (v1)
1. 앞뒤 공백 제거
2. 연속 공백 1개로 축소
3. 모든 공백 제거
4. 특수문자 제거 (`-`, `_`, `.`, `(`, `)`, `/`)
5. 한글/영문 소문자화(영문만 해당)
6. 동의어 치환 사전 적용

## 동의어 치환 사전
- `컴퓨터공학부` -> `컴퓨터공학과`
- `컴공` -> `컴퓨터공학과`
- `전자전기공학과` -> `전기전자공학과`
- `경영학부` -> `경영학과`
- `기계공학부` -> `기계공학과`

## 예시
- 입력: `컴 공` -> 정규화: `컴퓨터공학과`
- 입력: `컴퓨터공학부` -> 정규화: `컴퓨터공학과`
- 입력: `전기/전자공학과` -> 정규화: `전기전자공학과`

## 등록 처리 절차
1. 입력 학과명 수신
2. 정규화 함수 적용
3. `school_id + normalized_name` 중복 조회
4. 존재하면 기존 `department_id` 반환
5. 없으면 신규 생성

## 운영 병합 규칙
- 유사명으로 분리된 기존 학과가 발견되면 운영자가 수동 병합
- 병합 시 클릭 수는 합산, 이전 ID는 `alias` 테이블로 남김 (후속)
- MVP에서는 alias 테이블 없이 운영 문서로 기록 후 수동 조정
