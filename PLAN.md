# 전국 대학 교수 과제 압박 랭킹 서비스 콘셉트 설명서

## Summary
이 서비스는 학생들이 과제를 하며 느끼는 부담을 풍자형 게임으로 표현하는 모바일 웹 서비스다. 학생은 교수 프로필을 등록하거나 기존 교수 프로필을 찾아 클릭하고, 누적 클릭 수가 높아질수록 교수 캐릭터 등 위에 학생 캐릭터가 쌓인다. 핵심 재미는 `과제를 많이 내주는 교수일수록 더 무거워 보인다`는 시각적 과장에 있고, 결과적으로 전국 대학 단위의 교수 랭킹을 만든다.

서비스 톤은 공격적 폭로가 아니라 밈 기반 캠퍼스 풍자 게임으로 잡는다. 다만 실제 식별 리스크를 낮추기 위해 교수 노출 방식은 `학교 초성 + 학과 + 이름 초성`을 기본 포맷으로 사용한다. 예: `ㅇㅅ대 / 경영학과 / ㄱ교수`.

초기 버전은 모바일 웹으로 출시한다. 학생이 과제를 하다가 바로 접속해 클릭하는 사용 맥락이 핵심이므로 설치형 앱보다 접근성이 우선이다.

## Key Changes
### 제품 구조
- 홈: 실시간 전국 랭킹, 학교별 인기 교수, 오늘 가장 많이 눌린 교수 노출
- 검색/등록: 학생이 교수 프로필을 검색하고 없으면 새로 등록
- 교수 상세: 아바타, 소속 정보, 누적 클릭 수, 현재 쌓인 학생 수, 최근 클릭 추이 표시
- 랭킹: 전국 랭킹, 학교별 랭킹, 학과별 랭킹 분리 제공
- 내 활동: 내가 오늘 누른 교수, 즐겨찾기 교수, 공유 링크 제공

### 핵심 게임 규칙
- 순위 기준은 `순수 클릭 수`다
- 학생은 과제를 하다가 힘들 때마다 해당 교수를 클릭한다
- 클릭 수 1,000회당 학생 캐릭터 1명이 교수 등 위에 추가로 쌓인다
- 클릭 수가 늘수록 교수 아바타의 표정, 자세, 배경 효과가 점점 더 힘들어 보이게 변한다
- 전국 단위로 누적 클릭 수를 합산해 `대한민국 전국 대학교 과제로 학생을 힘들게 한 교수 랭킹`을 만든다
- 댓글, 자유게시판, 장문 리뷰는 넣지 않고 `클릭만`으로 참여를 단순화한다

### 교수 프로필/아바타 정책
- 노출 필드: 학교 초성, 학과명, 이름 초성
- 등록 필드: 학교명, 학과명, 교수 실명, 별칭, 수업 특징 키워드
- 공개 시 실명 전체는 숨기고 내부 식별값으로만 사용
- 아바타는 학생이 입력한 특징 키워드 기반으로 AI 생성
- 아바타 스타일은 실사보다 `캐릭터형 풍자 아바타`로 고정해 직접적 초상권 리스크를 낮춘다
- 동일 교수 중복 등록 방지를 위해 학교/학과/실명 기준 내부 중복 체크를 둔다

### 사용자 흐름
- 로그인: 소셜 로그인
- 첫 진입: 서비스 설명, 밈성 있는 예시 화면, 랭킹 미리보기
- 참여: 교수 검색 → 교수 선택 → 클릭
- 신규 등록: 교수 정보 입력 → 특징 키워드 입력 → AI 아바타 생성 → 프로필 생성
- 확산: 교수 상세 페이지 링크를 친구에게 공유해 클릭을 모은다

## Public Interfaces / Product Entities
- `User`
  - 소셜 계정 ID, 닉네임, 학교 선택값, 활동 이력
- `ProfessorProfile`
  - 학교명, 학교 초성, 학과명, 교수 실명(비공개), 이름 초성(공개), 특징 키워드, AI 아바타
- `ClickEvent`
  - 유저 ID, 교수 ID, 클릭 시각, 유입 경로
- `RankingSnapshot`
  - 전국/학교/학과 기준 순위, 누적 클릭 수, 학생 스택 수
- `AvatarGenerationInput`
  - 외형/말투/수업 스타일/과제 특징 키워드
- 핵심 화면 인터페이스
  - 홈 랭킹 화면
  - 교수 검색/등록 화면
  - 교수 상세 화면
  - 학교별 랭킹 화면
  - 공유 유입 랜딩 화면

## Test Plan
- 학생이 교수 프로필을 검색하고 기존 프로필에 클릭할 수 있어야 한다
- 학생이 없는 교수를 새로 등록하고 즉시 클릭 대상에 추가할 수 있어야 한다
- 클릭 수가 누적되면 랭킹이 전국/학교/학과 기준으로 즉시 반영되어야 한다
- 클릭 수 1,000 단위 도달 시 학생 스택 수가 정확히 1씩 증가해야 한다
- 교수 상세 화면에서 누적 클릭 수와 시각 효과가 동일한 기준으로 노출되어야 한다
- 동일 교수의 중복 등록 시 내부적으로 병합 또는 등록 제한 처리가 되어야 한다
- 아바타 생성 시 입력 키워드가 없거나 모호해도 기본 캐릭터 템플릿으로 대체 가능해야 한다
- 모바일 웹에서 로그인, 검색, 클릭, 공유까지 한 손 사용 흐름이 깨지지 않아야 한다

## Assumptions
- 이 문서는 정식 PRD가 아니라 이후 상세 기획서로 확장하기 위한 상위 개념 설명서다
- 서비스 톤은 `풍자형 캠퍼스 게임`이며, 정보성 플랫폼이나 수업평가 서비스로 포지셔닝하지 않는다
- 공개 범위는 전국 공개다
- 참여 방식은 소셜 로그인 기반이다
- 참여 기능은 클릭만 제공하고 댓글/게시판은 제공하지 않는다
- 랭킹은 객관적 과제량 검증이 아니라 학생 감정의 집단적 표현이라는 전제를 가진다
- 운영정책은 사용자 선택대로 `최소 운영개입`을 기본으로 두되, 실제 서비스화 단계에서는 허위 등록·명예훼손·초상권 이슈 때문에 법무 검토와 최소한의 신고/비공개 정책이 사실상 필요하다
